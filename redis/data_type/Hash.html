# Hash底层实现原理

# ziplist / listpack （数组）

当一个 hash对象只包含「少量键值对」且每个键值对的kv「要么就是小整数要么就是长度比较短的字符串」，那么它用 ziplist 作为底层实现，相关的参数是hash-zipmap-max-entries。

在 Redis 7.0 中，ziplist已经废弃了，交由 listpack 数据结构来实现了。

重点是下面的dict。

# dict：字典 哈希表

底层是个Hash表，源码有两个hash，一个是为了扩容而存在的。

// dict字典
typedef struct dict {
    // 这是个一维数组 元素为 dictEntry的指针
    dictEntry **table;
    //类型特定函数
    dictType *type;
    unsigned long size; //hash 表的大小
    unsigned long sizemask; // mask 计算索引值

} dict;
// dict内部的键值对
typedef struct dictEntry { 
    void *key;
    union {
        void *val;
        uint64_t u64;
        int64_t s64;
        double d;
    } v;
    struct dictEntry *next; // 解决哈希冲突，链式存储
} dictEntry;

# 解决哈希冲突：链式存储

发生哈希冲突时，也是用链表存储的，这和Java的HashMap一样，但不会转化为红黑树

# 扩容实现原理

当元素数量等于数组长度时就会进行扩容操作。

生成 RDB 和重写 AOF的过程中，不允许rehash

int dictExpand(dict *d, unsigned long size)
{
    dictht n; 
    // 扩容后的状态
    unsigned long realsize = _dictNextPower(size);
    // 初始化扩容后的字典元数据
    n.size = realsize;
    n.sizemask = realsize-1;
    n.table = zcalloc(realsize*sizeof(dictEntry*));
    n.used = 0;
    // 第一次初始化
    if (d->ht[0].table == NULL) {
        d->ht[0] = n;
        return DICT_OK;
    }
    // 放入新 ht[1] 中
    d->ht[1] = n;
    // 打上 rehash 标记
    d->rehashidx = 0; 
    return DICT_OK;
}

# 缩容触发时机

当字典的使用容量不足总空间的 10% 时（负载因子小于0.1），就会触发缩容，同样打上rehash标记

# 关键：渐进式rehash

我们知道触发了扩容缩容仅仅是打上 rehash 标记并做初始化，什么时候真正执行rehash呢？有两个触发时机：我们依次来看，先对初始化做一些必要的说明。

# 0、初始化

会同时保留两个新旧hash结构，新hash的大小为「大于等于使用量的二的幂」

同时维护一个 rehashIndex 变量，表示现在rehash到了数组的哪里，每次rehash的单位为桶，即一个bucket，随后rehashIndex+1；

# 1、定时任务

定时进行一次rehash，如每 100ms/ 次

每次执行时长不会超过 1ms

# 2、发生CRUD操作

每次发生CRUD操作，进行一次rehash。

# rehash具体行为

调用 dictRehash 函数，dictRehash 函数会把 ht[0] 中的元素依次添加到新的 Hash 表 ht[1] 中

# rehash期间的操作

  • 新增操作，都只会在ht[1]中发生；
  • 查询操作，会先去ht[0]中查找，然后再到ht[1]中查找；
  • 删改同查询。

# dict vs Java-HashMap

对于redis而言，内存空间十分宝贵，因此主要是为了节省空间，为此:

  • dict有缩容，HashMap没有
  • dict链表不会转化为红黑树，HashMap会
  • Redis解决哈希冲突的链式存储采用**「头插法」**，HashMap曾经也是，但后来改成了尾插法，这是因为Redis单线程不需要担心死锁问题。

同时，因为redis单线程的缘故，为了避免服务暂停阻塞，采用渐进式Hash

# Hash唯一约束：键值对个数

Hash的键值对个数上限是：4,294,967,295 (2^32 - 1)
